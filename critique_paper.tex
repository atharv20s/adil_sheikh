\begin{abstract}
This paper presents a formal critique and enhancement of the blockchain consensus framework proposed by Sheikh et al. (2020) for Electric Vehicle (EV) energy trading. While the original work successfully derives a static security threshold against False Data Injection (FDI) attacks, it crucially omits the temporal exposure introduced by the latency of the chosen OM($m$) consensus algorithm. We mathematically prove that OM($m$)'s $\mathcal{O}(n^m)$ message complexity creates an astronomical latency delay, subjecting the network to severe, repeated temporal attack windows. To resolve this, we introduce the Static-Temporal Security Framework (STSF) alongside Tower BFT, an $\mathcal{O}(n)$ pipeline that definitively seals the vulnerability window while retaining the original cryptographic integrity.
\end{abstract}
