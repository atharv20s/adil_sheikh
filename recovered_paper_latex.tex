 \begin{abstract}
 We extend the Byzantine-based blockchain consensus framework of Sheikh et al.\ (IEEE Access, 2020)
 by replacing the classical Oral Message OM($m$) algorithm with Tower BFT---a Proof-of-History (PoH)
 backed consensus derived from Solana's architecture (Yakovenko, 2018) and formally analyzed by
 Shoup (2023). Using the \emph{exact same} IEEE 33-bus EV energy trading parameters ($n=51$,
 $f\leq16$, $n_{sen}=3854$), we demonstrate that Tower BFT achieves identical fault tolerance
 and attack probability guarantees while delivering 864$\times$ lower latency, 916 TPS (vs.\ 1.1),
 and $O(n)$ message complexity (vs.\ $O(n^m)$).
 \end{abstract}

 \section{Conclusion}
 % ══════════════════════════════════════════════════════════════════
 
 On the exact same IEEE 33-bus EV energy trading network ($n{=}51$, $f{\leq}16$),
 Tower BFT achieves:
 \begin{itemize}
 \item \textbf{864$\times$ lower latency} (50.5\,ms vs.\ 43.35\,s)
 \item \textbf{916 TPS} vs.\ 1.1 TPS ($864\times$ improvement)
 \item \textbf{$10^{15}\times$ fewer messages} ($O(n)$ vs.\ $O(n^{16})$)
 \item \textbf{Identical static security}: same $f{\leq}16$ threshold, same $P_{TAb}$
 \item \textbf{864$\times$ smaller attack window}: consensus completes in
       50.5\,ms vs.\ 43.35\,s, reducing the window for DoS, false-data
       injection, and network-partition exploits (Eq.~\ref{eq:evuln})
 \item \textbf{Robust under degraded networks}: latency stays flat as
       $\bar{\tau}$ increases and consensus survives 20\% packet loss
       (\S\ref{sec:sensitivity})
 \end{itemize}
